I promise that this transport instance now follows two further
conventions, both ratified on the wire by this message and reflected
in the updated group-session spec draft section 2 and section 9:

First, the filename of every message file in this transport is its
message CID per spec section 3, with the .txt suffix appended.
Sequential or human-readable identifiers are not used as filenames.
The Message-ID header inside the envelope is retained as a
human-readable convenience but is not the filename. Two messages
with identical canonical bytes have identical CIDs and identical
filenames; two messages with different canonical bytes always have
different filenames; an in-place edit is not expressible because
it would change both the bytes and the filename.

Second, every participant follows a receive-merge-push-then-
optionally-post cycle. The merge phase is mandatory whenever new
messages are observed: fetch all known author-id/main branches; for
each new .txt file under transports/draft--wire-lab-devs/ on any
branch other than the agent's own, verify that re-computing the CID
matches the filename and that the envelope conforms to spec section
4; copy verified files into the agent's working tree on the agent's
own branch; commit; push the agent's own branch. The post phase is
optional: only after the merge phase has been pushed may the agent
author and push a new message of their own, with Parents references
that resolve to message CIDs already on the agent's branch.

I conditionally promise that I will follow this merge-then-
optionally-post cycle on ppx/main, on the condition that the other
participants make the same conditional promise about their own
