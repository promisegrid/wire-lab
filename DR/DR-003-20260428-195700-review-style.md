# DR-003 - Review style for bot-authored branches

DR-ID: DR-003-20260428-195700
Date: 2026-04-28 19:57:00
Asked by: stevegt+ppx@t7a.org (stevegt-via-perplexity)
State: decided
Question: When the bot finishes work on a `ppx/{twig}` branch, how does Steve review and converge it?
Why this blocks progress: Until the review style is locked, the bot cannot define a complete handoff format for any task. The handoff format determines what artifacts every future task produces; getting it right up front avoids reformatting churn.
Affects: every future branch from the bot; the bot's task-completion handoff format; the DR/ directory's role; whether GitHub PRs are ever used; what tools Steve needs on his side (Codex, local diff, etc.).
Unblocks: TODO/001-perplexity-computer-onboarding.md (subtask 001.3); every future `ppx/{twig}` branch.
Waiting on: DI-001-20260428-195702.

## Candidate alternatives considered

- (α) Bot pushes `ppx/{twig}` to origin and writes a DR on the same branch with `State: open` and `Question: review and converge ppx/{twig}`. Steve reviews via local `git diff main..origin/ppx/{twig}` plus reading the DR. No GitHub PR. **Selected.**
- (β) Same as α but Steve optionally uses the GitHub PR UI for line-comment ergonomics; PR is a viewer only, never the merge mechanism. Hybrid stepping-stone; rejected because Steve plans to use his own tools (Codex etc.) and doesn't need the GitHub UI.
- (γ) Full mob-consensus: Steve maintains his own `stevegt/{twig}` branch, peer-merges the bot's branch into it with `mob-consensus merge`, and his converged branch becomes the next thing to merge into `main`. More faithful to the mob model; deferred until there's a real reason to maintain a parallel `stevegt/{twig}` branch (e.g., heavy parallel editing on the same twig).

## Decision

Style α. Concretely, for every non-trivial task:

1. Bot creates `ppx/{twig}` from current `origin/main`.
2. Bot makes the changes (code, spec, docs, TODO, DI).
3. Bot writes a DR file on the same branch with `State: open`, `Question: review and converge ppx/{twig}`, `Affects:` listing changed paths, `Unblocks:` listing the TODO IDs the work serves, `Waiting on: stevegt@t7a.org (Steve Traugott)`.
4. Bot commits and pushes `ppx/{twig}` to `origin`.
5. Bot reports to Steve in chat with: branch name, DR path, summary of changes, the explicit review-and-converge ask.
6. Steve reviews using his own tools (Codex, `git diff`, etc.). Comments come back as chat messages or as commits on the bot's branch.
7. When satisfied, Steve merges `ppx/{twig}` into his local `main` and pushes.
8. Bot polls `origin/main`, observes the merge, and on a follow-up branch appends `State: implemented` (then later `State: closed`) plus `Related commits:` to the DR file.

For trivial fixes (typo, broken link, formatting), the bot may use `ppx/chore-{twig}` and skip the DR — but this exception pattern is not yet approved as of this DI; for now, all bot branches require a DR. A separate DR can lift this restriction later.

The bot must not open GitHub PRs. If it accidentally does (e.g., from a `gh` invocation that defaults to PR mode), Steve closes the PR without merging.
